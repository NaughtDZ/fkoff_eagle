#弃坑Eagle图片管理软件的PY脚本
Eagle固执又流氓的采取全复制逻辑进行图片管理，并且在10万以上图片的资源库跳转，性能极其堪忧同时竟然不具备跨资源库搜索筛选功能。总之烂中烂。
当你想放弃Eagle去导出已经按文件夹目录整理好的资源库，你发现了一个尴尬问题，你的500G的硬盘的资源库是300G，然后沟槽的Eagle全复制逻辑让你根本无法导出资源库图片。
那么。你就需要这个脚本了。

##第一步：准备工作（开启 Eagle API）
你需要先在 Eagle 软件里开启“开发者选项”以获取 API Token：

打开 Eagle。
点击菜单栏 Eagle > 偏好设置 > 开发者选项。
勾选 开启开发者工具。
复制那个 Master Token（访问令牌）。

##第二步：Python 脚本（API + 物理移动）
请确保你安装了 requests 库（用于调用 API）：

```<BASH>
pip install requests
```
然后运行本项目脚本（记得修改脚本内的路径！！！）

#这个脚本的执行逻辑：
连线内应 (Online)：脚本首先利用 Token 访问 Eagle 的本地 API，把你的整个目录树结构（ID <-> 文件夹名）完整地爬取下来，保存在 Python 的内存字典里。
断绝后路 (Offline)：爬取成功后，脚本会暂停，并提示你“请关闭 Eagle 软件”。
为什么这么做？ 就像你说的，空间不足。如果在 Eagle 开着的时候移动它的文件，虽然 Python 的 shutil 比较强力，但 Windows 可能会因为文件句柄占用报错，或者 Eagle 发现文件没了疯狂报错。关掉它是最稳妥的。
大迁徙 (Moving)：等你按回车确认后，脚本利用内存里的地图，开始深入每一个 .info 文件夹，抓取 metadata.json 里的 ID，匹配地图，然后执行 shutil.move。

#注意事项：
Token 必须正确：如果 Token 填错，第一步就会报错退出。
同盘符操作：请务必保证 EAGLE_LIBRARY_PATH (源) 和 TARGET_EXPORT_PATH (目标) 在同一个盘符（例如都在 D 盘）。只有这样，操作系统才会执行指针修改（耗时 0.01秒/文件，不占空间），而不是复制+删除（耗时久，且需要临时双倍空间）。
